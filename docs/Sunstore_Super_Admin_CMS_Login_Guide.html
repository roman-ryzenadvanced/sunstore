This guide shows how to access the Super Admin CMS (Central platform) and what to do on your first login.

Login steps (Super Admin CMS)

  1. Open the Central login page (Production or Preview): /central/login
  2. Enter your platform username and password, then press “Sign in”. You’ll be redirected to the dashboard.
  3. If the page loads but sign-in fails on Vercel: the CMS login requires the backend API to be reachable to issue a token.
Storefront vs CMS

The storefront can run with embedded preview/mock products. The Super Admin CMS needs the backend API for authentication and real data.

When Vercel is “frontend-only”

If you did not deploy the backend, the Central login endpoint won’t exist, so CMS login won’t work until the backend is online.

Initial master Super Admin login

On a full-stack deployment (backend + database), the backend can create the first master Super Admin account via bootstrap variables:

SUPER_ADMIN_BOOTSTRAP_USERNAME
SUPER_ADMIN_BOOTSTRAP_PASSWORD

Development default

If the backend is running with APP_ENV=development and you did not set the bootstrap variables, the app auto-creates a default master Super Admin:

Username: admin
Password: changeme123

Important: this default is for development/testing only. For production, set bootstrap credentials explicitly and rotate them immediately after first login.

First actions after login

After you reach the Central dashboard, typical first steps are:

  • Create a new Store (site): choose slug, template, and owner credentials.
  • Review Products: cross-store listing and per-site catalogs.
  • Review Orders: monitor payments and statuses (demo mode is supported).
  • Configure Email: set platform-level SMTP/Gmail and optionally override per site.
  • Set custom domain instructions for the store owner (if used).

Troubleshooting

  • Login page loads but sign-in fails: confirm the backend API is up and reachable from the frontend.
  • Storefront shows products but CMS is empty: storefront can use embedded preview data; CMS needs backend.
  • Forgot password: rotate via database change or re-bootstrap on a fresh DB.
  • Payments: set TBANK_MODE=demo for safe testing; use sandbox/live only with real credentials.

Security notes

  • Change any default credentials immediately (especially admin / changeme123).
  • Use HTTPS for production, and restrict admin/CMS access where possible.
  • Do not commit real secrets into the repo; store them in your deployment environment.
